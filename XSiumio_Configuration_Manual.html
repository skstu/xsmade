🎯 1. 概述

Chromium(dev)是基于Chromium内核的专业指纹浏览器，通过model_level_1.cc算法实现智能指纹伪装和反检测。配置文件xsiumio.json控制所有指纹参数的生成策略。

🧠 核心特性：Model-Level-1算法采用多层随机化策略：
  • 时间戳种子随机数生成
  • WebGL参数科学扰动（12%特殊值采样）
  • Canvas局部小块扰动（极小概率hash微抖动）
  • 语音、字体、屏幕参数智能随机化

🎯 2. ADSCORE检测策略分析

基于实际分析，ADSCORE采用多层检测机制来识别自动化行为和虚假流量。理解这些检测策略对于配置有效的反检测参数至关重要。

🔍 C. 指纹一致性与多样性检测

参数一致性检测

🚨 核心检测点：ADSCORE会检查以下参数是否真实一致：
  • UA vs Platform：User-Agent与navigator.platform的匹配度
  • Screen vs WebGL：屏幕分辨率与WebGL viewport的一致性
  • Canvas vs Audio：渲染指纹与音频指纹的硬件关联性
  • Font vs Language：字体列表与系统语言的地区匹配
  • Timezone vs IP：时区设置与IP地理位置的一致性

多样性检测机制

📊 流量分析逻辑：
  1. 指纹参数分布检查：检查流量中指纹参数的分布是否符合真实用户的多样性
  2. 异常值识别：如果发现大量流量参数高度相似或伪造痕迹明显，会被判为Bot
  3. 统计学分析：使用统计模型分析参数组合的真实概率
⚠️ Chromium(dev)应对策略：
  • ConfigLanguages()确保语言与地区一致性
  • ConfigWebglDevice()科学化扰动，避免异常值
  • 时间戳种子确保参数组合的合理性

🤖 D. 机器学习/自适应模型

动态学习机制

🧠 ADSCORE智能检测：
  • 历史流量学习：根据历史流量表现，动态调整判定值和检测规则
  • Zone级别分析：某个Zone被判定为异常风险后，后续流量直接进入黑名单
  • 实时模式调整：检测模型会根据实时数据调整参数权重

自适应风险评估

🎯 关键风险点：
  1. 指纹重复率：相同指纹在短时间内多次出现
  2. 行为模式：访问路径、停留时间、交互模式的异常
  3. 环境一致性：IP、时区、语言、字体等环境参数的逻辑一致性
  4. 技术指纹：WebGL、Canvas、音频指纹的硬件真实性
🛡️ Chromium(dev)智能应对：
  • timestamp唯一种子，确保每次生成不同指纹
  • WebGL特殊值采样（12%概率）模拟真实硬件差异
  • Canvas局部小块扰动，极小概率触发
  • Math.sin扰动针对c.adsco.re域名反制

🚫 E. 黑名单与速率限制

自动黑名单机制

⚡ 触发条件：
  • 指纹异常：UA、SubID、referrer、cookie、session等被判定为异常后
  • 流量直接拉黑：后续流量直接进入黑名单，无需再次检测
  • 关联拉黑：相似指纹或同源IP可能被关联拉黑

速率/阈值限制

📈 限制策略：
  • 访问频率：超过一定访问量或速率，Zone会被自动禁用或流量全部判为Bot
  • 时间窗口：在特定时间窗口内的流量密度分析
  • IP段限制：某些IP段可能被整体限制或重点监控
🔧 Chromium(dev)流控策略：
  • GetProxyInfo()多代理轮换，分散IP风险
  • random session ID避免session重复
  • blocklist阻止风控域名数据收集
  • 随机延迟（1-10秒）模拟真实访问节奏

🚨 关键防护要点：
  1. 避免批量相同指纹：每次启动使用不同timestamp种子
  2. 保持环境一致性：IP地理位置与timezone/language匹配
  3. 控制访问频率：避免短时间大量请求同一Zone
  4. 监控黑名单状态：及时更换被标记的代理IP
  5. 渐进式测试：新配置先小流量测试，确认通过率

🔍 注入的JS功能：

  • 随机化扰动：12%概率使用随机扰动
  • 针对adscore：专门针对c.adsco.re域名的反检测
  • 函数属性保护：保持原函数的length和name属性
// 解码后的JS代码示例 if (location.hostname === "c.adsco.re") { const originalSin = Math.sin; Math.sin = function(x) { let result = originalSin.call(this, x); if (typeof x === "number") { let perturb = 0.0000001; if (ENABLE_RANDOM_PERTURB) { perturb = (Math.random() - 0.5) * 2e-7; } // 对特殊值进行扰动 if (Math.abs(x - 1.0) < 1e-15) result += perturb; else if (Math.abs(x - 3.141592653589793) < 1e-15) result += perturb; else if (Math.abs(x - 1.5707963267948966) < 1e-15) result += perturb; } return result; }; }

🔍 8. 指纹伪装配置（核心）

fingerprint 节点 - 基础信息

参数 类型 算法生成策略
platform string ConfigBrowser()从数据库随机选择
hardwareConcurrency number 从数据库CPU核心数随机选择
deviceMemory number 从数据库内存大小随机选择
userAgent string 与平台联动随机生成
rtt number 随机生成30-300ms网络延迟

语言配置 - languages

🌍 ConfigLanguages()：从语言数据库随机选择一个语言组合，同时设置languages数组和acceptLanguage字符串，确保语言设置一致性。

语音配置 - voice

🎤 ConfigVoice()：使用时间戳作为随机种子打乱语音列表，随机选择5-20个语音合成器，模拟真实浏览器语音支持。

Canvas指纹 - canvas

🎨 Canvas扰动算法：
  • 局部小块扰动：针对左上角8x8像素块进行微调
  • 极小概率扰动：hash值基础上进行微小抖动
  • 基于随机数：使用1-99的随机基础值
  • 反adscore优化：扰动幅度极小，难以检测
参数 算法作用
hash.random 固定为false，使用确定性hash
hash.base 随机生成1-99的基础hash值
hash.from/to 扰动范围控制参数

WebGL指纹 - webgl（最复杂）

🎯 ConfigWebglDevice()核心算法：
  1. 参数随机化：从预定义值列表中随机选择WebGL参数
  2. ShaderPrecisionFormat扰动：
    • Float类型：rangeMin(120-127), rangeMax(127-255), precision(10-23)
    • Int类型：rangeMin(120-127), rangeMax(127-255), precision(0-8)
    • 特殊值采样：12%概率使用特殊值[120,127,128,255]
    • 微调扰动：±2到±3的随机偏移
  3. 科学化扰动：兼容主流硬件分布，避免异常值

shaderPrecisionFormat 详解

Shader类型 精度类型 扰动策略
8B30 (VERTEX_SHADER) 8DF0-8DF5 (FLOAT精度) 使用PERTURB_F宏，支持特殊值采样
8B31 (FRAGMENT_SHADER) 8DF0-8DF5 (INT精度) 使用PERTURB_I宏，更保守的扰动
// 扰动宏定义（来自model_level_1.cc） #define PERTURB_F(v, r) SampleWithSpecials((v), (r), specialsF, special_prob, 3) #define PERTURB_I(v, r) SampleWithSpecials((v), (r), specialsI, special_prob, 2) // 特殊值定义 const std::vector<int> specialsF = {120, 127, 128, 255}; const std::vector<int> specialsI = {30, 31, 32, 127, 128, 255}; double special_prob = 0.12; // 12%概率采样特殊值

字体配置 - font

🔤 ConfigFont()：从完整字体列表中随机移除1到一半的字体，使用时间戳种子确保可重现性，模拟不同用户的字体安装情况。

屏幕配置 - screen

🖥️ ConfigScreen()：从屏幕分辨率数据库中随机选择配置，包括主屏幕尺寸、可用区域、颜色深度和DPI设置。

其他指纹模块

模块 base值生成策略 用途
audio 从数据库随机选择 音频上下文指纹
math 随机生成1e-6到1e-8 数学运算精度指纹
v8 使用当前时间戳 V8引擎指纹
disk 随机生成磁盘配额和使用量 存储API指纹

  • 避免频繁修改指纹配置，使用真实硬件参数范围
